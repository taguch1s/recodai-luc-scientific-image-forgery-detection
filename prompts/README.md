# Kaggle調査プロンプト & ワークフロー

NFL Big Data Bowl 2026の競技において、効果的な情報収集・分析・戦略立案を行うためのプロンプト集とワークフローです。

## 📁 ファイル構成

```
prompts/
├── kaggle_analysis_prompts.md    # 基本的な分析用プロンプト集
└── kaggle_workflow_prompts.md    # ワークフロー特化プロンプト集

analysis/
├── research_summary_template.md  # 調査結果まとめテンプレート
└── tracking_sheets.md           # トラッキング用スプレッドシート
```

## 🚀 クイックスタート

### 1. 初期調査フェーズ
```bash
# Step 1: コンペ概要把握
prompts/kaggle_workflow_prompts.md の "Step 1: 初期情報収集プロンプト" を使用

# Step 2: トップNotebook分析
prompts/kaggle_analysis_prompts.md の "基本分析プロンプト" を使用
```

### 2. 詳細分析フェーズ
```bash
# 有望なNotebookの詳細分析
prompts/kaggle_analysis_prompts.md の "詳細技術分析プロンプト" を使用

# Discussion要約
prompts/kaggle_analysis_prompts.md の "Discussion要約プロンプト" を使用
```

### 3. 戦略立案フェーズ
```bash
# 手法比較・戦略立案
prompts/kaggle_analysis_prompts.md の "手法比較プロンプト" を使用
prompts/kaggle_workflow_prompts.md の "Step 4: 手法比較・戦略立案プロンプト" を使用
```

## 📋 使用手順

### Phase 1: 情報収集 (Week 1)

1. **コンペ基本情報収集**
   - `kaggle_workflow_prompts.md` の Step 1 プロンプトを使用
   - `research_summary_template.md` の基本情報セクションを埋める

2. **トップNotebook分析**
   - リーダーボード上位者のpublic notebookを3-5個選択
   - `kaggle_analysis_prompts.md` の基本分析プロンプトで分析
   - `tracking_sheets.md` でトラッキング開始

3. **重要Discussion特定**
   - 高評価・多コメントのdiscussionを5-10個選択
   - Discussion要約プロンプトで分析

### Phase 2: 詳細分析 (Week 2)

1. **技術的深掘り**
   - 有望なnotebookを詳細技術分析プロンプトで分析
   - NFL特化プロンプトも併用

2. **手法の分類・整理**
   - 手法比較プロンプトで統合分析
   - `tracking_sheets.md` の手法実装トラッキングを更新

### Phase 3: 戦略立案 (Week 2-3)

1. **実装戦略策定**
   - Step 4 戦略立案プロンプトで総合分析
   - `research_summary_template.md` の実装計画セクションを完成

2. **リスク評価**
   - 技術的・スケジュール的リスクを評価
   - バックアップ計画を策定

### Phase 4: 継続的更新

1. **週次監視**
   - Step 5 追跡・更新プロンプトで新着情報チェック
   - `tracking_sheets.md` で進捗管理

2. **戦略調整**
   - 新しい情報に基づく戦略修正
   - 実装優先度の再評価

## 🎯 プロンプト選択ガイド

### 状況別推奨プロンプト

| 状況             | 推奨プロンプト          | 所要時間   |
| ---------------- | ----------------------- | ---------- |
| コンペ開始直後   | Step 1: 初期情報収集    | 1-2時間    |
| Notebook詳細分析 | 基本分析 → 詳細技術分析 | 30-60分/個 |
| Discussion調査   | Discussion要約          | 15-30分/個 |
| 手法比較検討     | 手法比較プロンプト      | 2-3時間    |
| 戦略立案         | Step 4: 戦略立案        | 2-3時間    |
| 定期アップデート | Step 5: 追跡・更新      | 30分/週    |

#### 品質確保
- 複数の観点から評価
- 定量的指標を併用
- 定期的な見直し

#### チーム作業
- テンプレートで統一性確保
- 役割分担で効率化
- 定期的な情報共有

## 📊 アウトプット管理

### 必須成果物
1. **調査サマリー** (`research_summary_template.md`)
   - コンペ概要、主要発見、推奨戦略

2. **トラッキングシート** (`tracking_sheets.md`)
   - Notebook/Discussion分析状況
   - 手法実装進捗
   - 週次進捗管理

3. **戦略書**
   - 実装計画とスケジュール
   - リスク管理計画
   - バックアップ戦略

### 品質基準
- **完全性**: 重要な情報の見落としがない
- **正確性**: 技術的評価が適切
- **実用性**: 実装に直結する内容
- **更新性**: 継続的なメンテナンスが可能

## 🔧 カスタマイズ

### コンペ特化カスタマイズ
- NFL特化プロンプトを参考に、他のドメインに応用
- 評価基準をコンペの特性に合わせて調整
- 必要な分析観点を追加

### チーム向けカスタマイズ
- 役割分担を明確化
- 情報共有フローを確立
- 品質チェック体制を構築

## 📈 効果測定

### 成功指標
- **調査効率**: 時間当たり分析Notebook数
- **情報品質**: 実装に結びついた手法の割合
- **戦略精度**: 実際のスコア改善への貢献度
- **時間短縮**: 従来手法との比較

### 改善サイクル
1. 実施結果の振り返り
2. プロンプトの改善
3. ワークフローの最適化
4. 次回への反映

## 💡 ベストプラクティス

### 調査時のポイント
- **早期開始**: コンペ開始と同時に調査開始
- **継続更新**: 週1回の定期アップデート
- **深さと広さ**: トップ解法の詳細分析 + 幅広い手法調査
- **実装指向**: 調査結果を実装計画に直結

### よくある落とし穴
- 調査に時間をかけすぎて実装が遅れる
- 表面的な分析で本質を見逃す
- 新しい手法に飛びつき基本を疎かにする
- 情報の整理・更新を怠る

## 🆘 トラブルシューティング

### Q: 分析に時間がかかりすぎる
A: 高速スクリーニングプロンプトで事前フィルタリング。優先度の高いものから着手。

### Q: 技術的内容が理解できない
A: 基本分析プロンプトから開始。段階的に詳細分析に移行。

### Q: 戦略立案ができない
A: 手法の分類・整理から開始。比較表を作成して可視化。

### Q: 継続的な更新ができない
A: 週次チェックリストを作成。アラート設定で確実に実施。

# Kaggle調査ワークフロー用プロンプト

## Step 1: 初期情報収集プロンプト

```
NFL Big Data Bowl 2026のKaggle情報を調査します。以下の観点で整理してください：

### 基本情報整理
1. **コンペ概要**
   - 問題設定とタスク詳細
   - 評価指標
   - データセット構成
   - 提出形式

2. **現在の状況**
   - 参加チーム数
   - リーダーボード上位の特徴
   - 公開ノートブック数とトレンド
   - Discussionでの主要話題

3. **技術トレンド**
   - 主流となっているアプローチ
   - 新しく登場している手法
   - まだ試されていない可能性のある手法

**出力**: 調査サマリー（A4 1-2ページ）
```

## Step 2: ノートブック詳細分析プロンプト

```
選択したnotebookを詳細分析してください：

### 技術的深掘り分析

**URL/タイトル**: [ノートブック情報]

**分析フレームワーク**:

1. **問題アプローチ**
   ```
   - 問題の解釈方法
   - 前提条件・仮定
   - 解決方針の妥当性
   ```

2. **データ理解・処理**
   ```
   - データ探索の深さ
   - 前処理の妥当性
   - 特徴量エンジニアリングの質
   - ドメイン知識の活用度
   ```

3. **モデリング戦略**
   ```
   - アルゴリズム選択理由
   - ハイパーパラメータ戦略
   - 検証方法の適切性
   - 過学習対策
   ```

4. **実装品質**
   ```
   - コードの可読性・保守性
   - 実行効率性
   - 再現性
   - エラーハンドリング
   ```

5. **結果・評価**
   ```
   - スコア・順位
   - 改善量の定量化
   - 失敗例・エラー分析
   - 解釈可能性
   ```

6. **独創性・価値**
   ```
   - 新規性のある部分
   - 他手法との差別化点
   - 応用可能性
   - 学習価値
   ```

**スコアリング** (1-5点):
- 技術的完成度: ___/5
- 独創性: ___/5
- 実用性: ___/5
- 学習価値: ___/5
- 実装容易さ: ___/5

**総合評価**: ___/25点

**実装推奨度**:
- [ ] 必須実装
- [ ] 高推奨
- [ ] 検討価値あり
- [ ] 参考程度
- [ ] 不要

**出力**: 技術分析レポート
```

## Step 3: Discussion要約プロンプト

```
以下のDiscussionから重要な情報を抽出してください：

### Discussion分析テンプレート

**Discussion情報**:
- タイトル:
- 作成者:
- 投票数:
- コメント数:

**内容分析**:

1. **主要論点**
   - 議論の中心となっている問題・課題
   - 提起されている疑問・仮説

2. **提案された解決策**
   - 具体的な手法・アプローチ
   - コード例・実装方法
   - パラメータ・設定値

3. **コミュニティの反応**
   - 賛成・反対意見
   - 追加のアイデア・改善案
   - 実験結果・検証報告

4. **実用的価値**
   - 実装可能性
   - 効果の期待値
   - 適用可能な場面

**重要度評価**:
- データ理解: ★★★★★
- 手法改善: ★★★★★
- 実装参考: ★★★★★
- 戦略立案: ★★★★★

**アクションアイテム**:
- [ ] 実装検討
- [ ] 追加調査必要
- [ ] 実験計画立案
- [ ] 他資料との照合

**出力**: Discussion要約カード
```

## Step 4: 手法比較・戦略立案プロンプト

```
収集した複数の情報源から、最適な戦略を立案してください：

### 戦略立案フレームワーク

**入力情報**:
- 分析済みNotebook: [X]個
- 重要Discussion: [Y]個
- その他参考資料: [Z]個

**比較分析**:

1. **手法カテゴリ別整理**
   ```
   ベースライン手法:
   - 手法A: [効果][実装コスト][安定性]
   - 手法B: [効果][実装コスト][安定性]

   応用手法:
   - 手法C: [効果][実装コスト][安定性]
   - 手法D: [効果][実装コスト][安定性]

   先進的手法:
   - 手法E: [効果][実装コスト][安定性]
   - 手法F: [効果][実装コスト][安定性]
   ```

2. **パフォーマンス分析**
   ```
   効果/コスト分析:
   - 最も効率的: [手法名] (理由)
   - 最も高性能: [手法名] (理由)
   - 最も安定: [手法名] (理由)
   - 最も独創的: [手法名] (理由)
   ```

3. **実装戦略**
   ```
   Phase 1 (Week 1-2): ベースライン構築
   - 実装予定: [手法リスト]
   - 目標スコア: [数値]
   - 検証方法: [CV戦略]

   Phase 2 (Week 3-4): 改善・最適化
   - 実装予定: [手法リスト]
   - 改善観点: [特徴量/モデル/アンサンブル]
   - 目標改善: [数値]

   Phase 3 (Week 5-6): 高度化・統合
   - 実装予定: [手法リスト]
   - アンサンブル戦略: [方法]
   - 最終調整: [計画]
   ```

4. **リスク管理**
   ```
   技術的リスク:
   - [リスク1]: [対策]
   - [リスク2]: [対策]

   スケジュールリスク:
   - [リスク1]: [対策]
   - [リスク2]: [対策]

   バックアップ計画:
   - 主計画失敗時: [代替手法]
   - 時間不足時: [最優先実装]
   ```

**最終推奨**:
```
優先実装順位:
1. [手法名] - [理由] - [期待効果]
2. [手法名] - [理由] - [期待効果]
3. [手法名] - [理由] - [期待効果]

組み合わせ戦略:
- [手法A] + [手法B] = [期待シナジー]
- [手法C] + [手法D] = [期待シナジー]

成功確率の高いパス:
[具体的な実装順序とスケジュール]
```

**出力**: 実行可能な戦略書
```

```
**既存戦略の見直し**:
```


## 使用ガイドライン

### 効率的な調査順序
1. **Step 1**: コンペ全体の把握 (1-2時間)
2. **Step 2**: トップノートブック3-5個の詳細分析 (各30-60分)
3. **Step 3**: 重要Discussion 5-10個の要約 (各15-30分)
4. **Step 4**: 統合分析と戦略立案 (2-3時間)
5. **Step 5**: 定期的な情報更新 (週30分程度)

### 品質確保のポイント
- 主観的評価は根拠を明記
- 数値的評価は可能な限り定量化
- 実装可能性を常に考慮
- 最新情報への感度を保つ

### 出力形式の統一
- Markdown形式での構造化
- 図表は必要に応じて追加
- チェックリスト形式で進捗管理
- 日付・バージョン管理を徹底

# Kaggle競技調査用プロンプト集

## 1. Notebook分析用プロンプト

### 基本分析プロンプト
```
以下のKaggle notebookの内容を分析してください：

**分析項目：**
1. **概要・目的**
   - このnotebookの主な目的と解決しようとしている問題
   - アプローチの全体的な戦略

2. **手法・技術**
   - 使用されている主要なアルゴリズム・モデル
   - 特徴量エンジニアリング手法
   - データ前処理・クリーニング手法
   - 評価手法・メトリクス

3. **革新的・優秀な点**
   - 他と差別化されている独自のアプローチ
   - 技術的に優れている部分
   - 創造性のある解決方法

4. **コンペ核心度評価**
   - このアプローチがコンペの核心となる手法かどうか（5段階評価）
   - その理由と根拠
   - 他の解法への応用可能性

5. **実装可能性**
   - 実装の複雑さ（5段階評価）
   - 必要なリソース（計算時間、メモリ等）
   - 再現性の評価

**フォーマット：**
Markdownで構造化して出力してください。
```

### 詳細技術分析プロンプト
```
このKaggle notebookの技術的側面を詳細に分析してください：

**詳細分析項目：**

1. **データ処理パイプライン**
   - データローディング方法
   - 前処理ステップの詳細
   - データ変換・正規化手法
   - 欠損値処理方法

2. **特徴量エンジニアリング**
   - 作成された特徴量の種類と説明
   - ドメイン知識の活用方法
   - 特徴量選択手法
   - 特徴量の重要度分析

3. **モデリング戦略**
   - 使用モデルの選択理由
   - ハイパーパラメータ調整方法
   - アンサンブル手法（使用している場合）
   - クロスバリデーション戦略

4. **結果・パフォーマンス**
   - スコア・順位情報
   - 改善効果の定量的評価
   - エラー分析・失敗例の検討

5. **コード品質**
   - コードの可読性・構造
   - 計算効率性
   - 再利用可能性

**出力：**
技術レポート形式でまとめてください。
```

## 2. Discussion分析用プロンプト

### Discussion要約プロンプト
```
以下のKaggle discussionの内容を分析してください：

**分析観点：**

1. **討議内容の分類**
   - 問題理解・データ理解に関する議論
   - 手法・アルゴリズムに関する議論
   - 実装・技術的課題に関する議論
   - 結果・スコア改善に関する議論

2. **重要な洞察・発見**
   - コンペデータに関する重要な発見
   - 効果的なアプローチの提案
   - 落とし穴・注意点の指摘
   - ドメイン知識の共有

3. **コミュニティの反応**
   - 投票数・コメント数
   - 議論の活発さ
   - 合意形成の状況

4. **実用性評価**
   - 提案内容の実装可能性
   - 効果の期待値
   - 他参加者への影響度

**出力形式：**
- 要約：3-5文で概要
- 重要ポイント：箇条書き
- 実用度：★★★★★（5段階）
- 推奨度：コンペ攻略への有用性
```

## 3. 総合分析・比較プロンプト

### 手法比較プロンプト
```
複数のKaggle notebook/discussionから得られた情報を比較分析してください：

**比較分析項目：**

1. **手法の分類・整理**
   - アプローチの種類別分類
   - 使用技術のマッピング
   - 複雑さレベルでの階層化

2. **パフォーマンス比較**
   - スコア・順位での比較
   - 実装コスト vs 効果の分析
   - 安定性・再現性の比較

3. **トレンド分析**
   - 主流となっている手法
   - 新しく注目されている手法
   - 時系列での手法の変遷

4. **推奨戦略**
   - 初心者向け推奨アプローチ
   - 上級者向け推奨アプローチ
   - 最も有望な手法の組み合わせ

5. **実装優先度**
   - 効果/コスト比での優先度ランキング
   - 段階的実装プラン
   - リスク評価

**出力：**
戦略レポート形式（表・図を含む）
```

### コンペ戦略立案プロンプト
```
収集したKaggle情報を基に、効果的なコンペ戦略を立案してください：

**戦略立案要素：**

1. **現状分析**
   - 現在のリーダーボード状況
   - 主要な手法とその効果
   - 未開拓領域の特定

2. **差別化戦略**
   - 独自性を持てるアプローチ
   - 組み合わせによる新手法
   - リスク vs リターン分析

3. **実装ロードマップ**
   - Phase1: 基本実装（ベースライン確立）
   - Phase2: 改善・最適化
   - Phase3: 高度な手法・アンサンブル

4. **リソース配分**
   - 時間配分計画
   - 計算リソース使用計画
   - 学習・調査時間の確保

5. **リスク管理**
   - 実装失敗時のバックアップ計画
   - 締切に向けたタイムマネジメント
   - 技術的困難への対処法

**成果物：**
実行可能な戦略書（Markdown形式）
```

## 5. 情報収集効率化プロンプト

### 高速スクリーニングプロンプト
```
大量のKaggle情報から重要なものを効率的に特定してください：

**スクリーニング基準：**

1. **優先度判定（各項目1-5点）**
   - スコア改善への貢献度
   - 実装の容易さ
   - 独創性・新規性
   - 再現性・安定性
   - 計算効率性

2. **カテゴリ分類**
   - 必須実装（基本手法）
   - 推奨実装（効果大）
   - 検討実装（興味深い）
   - 参考のみ（学習用）

3. **1分間要約**
   - 手法の核心を1文で
   - 主要技術を箇条書きで
   - 期待効果を数値で

**出力：**
優先度付きリスト（Excel表形式でも出力可能）
```

## 使用方法

1. **個別分析**: 各notebookに対して基本分析プロンプトを適用
2. **詳細調査**: 有望なnotebookに対して詳細技術分析プロンプトを適用
3. **情報統合**: 総合分析プロンプトで戦略を立案
4. **効率化**: 大量情報処理時は高速スクリーニングプロンプトを使用

これらのプロンプトを状況に応じて組み合わせて使用することで、効果的なKaggle競技調査が可能です。
